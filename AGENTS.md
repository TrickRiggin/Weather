# Repository Guidelines

## Project Structure & Module Organization
Read `CLAUDE.md` before making changes; it captures the repo's current architecture, data sources, and operational gotchas.

`src/` contains the Vite + React frontend. `App.jsx` is the main UI, `main.jsx` boots the app, and `assets/` holds static images. Most other files in `src/` such as `markets.js`, `forecasts.js`, `edges.js`, `observations.js`, `results.js`, and `meta.js` are generated by the Python pipeline and should not be hand-edited unless you are changing the generator.

`refresh_weather.py` is the primary data pipeline. `calibrate.py` builds calibration data into `data/city_sigma.json`. Historical artifacts live under `data/`, with calibration caches in `data/calibration/`. Automation is in `.github/workflows/refresh.yml`.

## Build, Test, and Development Commands
Install frontend dependencies with `npm install`.

- `npm run dev`: start the local Vite dev server.
- `npm run build`: produce a production build in `dist/`.
- `npm run preview`: serve the built app locally.
- `npm run lint`: run ESLint across the frontend.
- `python refresh_weather.py`: run the pipeline in local/console mode.
- `python refresh_weather.py --write-data`: refresh generated `src/` and `data/` files.
- `python refresh_weather.py --backtest`: print historical performance from resolved signals.
- `python calibrate.py --report`: inspect cached calibration stats.

## Coding Style & Naming Conventions
Frontend code uses ES modules, React function components, and PascalCase for components. Existing JSX favors double quotes and semicolons; follow the file you are editing. Python uses 4-space indentation, snake_case, and small helper functions with direct docstrings. Keep manual edits out of generated JS output files unless the pipeline is being updated. Run `npm run lint` before opening a PR.

## Testing Guidelines
There is no dedicated automated test suite yet. Treat `npm run lint`, `npm run build`, and targeted Python script runs as the minimum validation set. When changing the pipeline, run `python refresh_weather.py` and, if outputs change intentionally, rerun with `--write-data`. When changing calibration logic, also run `python calibrate.py --report`.

## Commit & Pull Request Guidelines
Recent history uses short, imperative summaries for feature work and timestamped `Refresh: YYYY-MM-DD HH:MM UTC` commits for automation. Keep human-authored commits focused, descriptive, and separate from bulk refresh output when possible. PRs should state what changed, which commands were run, whether generated files changed intentionally, and include screenshots for UI changes.

## Security & Configuration Tips
Secrets are loaded from environment variables or local key files; never commit API keys. `.env` may exist locally, but production automation relies on GitHub Actions secrets such as `OPENMETEO_API_KEY`.
